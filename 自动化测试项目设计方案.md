# UI自动化测试项目设计方案

## 一、项目概述

### 1.1 背景
解决Web UI自动化测试中的滑块验证码问题，采用半自动登录+缓存复用的方案。

### 1.2 核心思路
- **登录阶段**：Selenium半自动化 + 人工介入处理验证码
- **缓存保存**：保存浏览器cookies、localStorage、token
- **测试复用**：后续测试直接加载已登录的浏览器状态

## 六、关键技术点

### 6.1 Chrome用户数据目录（关键）

```python
options = webdriver.ChromeOptions()

# 核心配置：指定用户数据目录
# 这会保存所有浏览器状态：cookies、localStorage、sessionStorage等
options.add_argument(f"--user-data-dir={user_data_dir}")

# 防止被检测为自动化
options.add_argument("--disable-blink-features=AutomationControlled")
options.add_experimental_option("excludeSwitches", ["enable-automation"])
options.add_experimental_option("useAutomationExtension", False)
```

### 6.2 检测登录状态的方法

```python
# 方法1：URL变化
if driver.current_url != login_url:
    # 登录成功

# 方法2：特定元素出现
if WebDriverWait(driver, 5).until(
    EC.presence_of_element_located((By.ID, "user-avatar"))
):
    # 登录成功

# 方法3：localStorage中有token
token = driver.execute_script("return localStorage.getItem('auth_token')")
if token:
    # 登录成功

# 方法4：cookie中存在session标识
cookies = driver.get_cookies()
if any(c['name'] == 'session_id' for c in cookies):
    # 登录成功
```

### 6.3 等待人工操作的实现

```python
def wait_for_manual_login(self, driver, timeout=300):
    """等待用户手动完成滑块验证"""

    print("⏳ 检测到滑块验证码，请手动完成验证...")

    start_time = time.time()
    while time.time() - start_time < timeout:
        # 检查localStorage中的token
        token = driver.execute_script(
            "return localStorage.getItem('auth_token')"
        )
        if token:
            print("✅ 检测到登录成功！")
            return True

        # 检查URL变化
        if "login" not in driver.current_url:
            print("✅ 检测到页面跳转，登录成功！")
            return True

        time.sleep(2)

    raise TimeoutError("等待手动登录超时")
```

## 十一、预期效果

✅ **首次运行**：自动输入账号密码 → 等待你手动滑动验证码 → 验证通过 → 保存缓存

✅ **后续运行**：检测到有效缓存 → 直接加载 → 跳过登录步骤 → 开始测试

✅ **缓存过期**：检测到缓存无效 → 自动重新执行半自动登录流程

## 十二、注意事项

1. **user-data-dir 冲突**：同一个目录不能被多个浏览器实例同时使用，测试时要注意单实例运行
2. **缓存安全性**：cache目录应加入.gitignore，避免提交敏感信息
3. **Token刷新**：如果token会过期，需要添加自动刷新机制
4. **多账号支持**：如果需要多账号，使用不同的session_name隔离缓存
5. **CI/CD兼容**：如果需要在CI环境运行，需要特殊处理（如远程桌面）
